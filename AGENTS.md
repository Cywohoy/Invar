# Instructions for Codex

- 작업 전에 `PROJECT.md`를 읽는다.
- 요구 범위를 넘어서는 기능을 구현하지 않는다.
- 아직 확정되지 않은 언어 문법을 임의로 결정하지 않는다.
- 새 의존성을 추가하기 전에 필요성을 설명한다.
- 모든 변경에는 테스트를 함께 추가한다.
- 작업 후 테스트와 빌드를 실행한다.
- 실행하지 못한 검증은 명시한다.
- 컴파일러 코어에서 DOM 및 Node.js 전용 API를 사용하지 않는다.
- TypeScript의 strict 모드를 유지한다.
- C++ 개발자가 이해할 수 있도록 생태계 특유의 결정을 설명한다.
